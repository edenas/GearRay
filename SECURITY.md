# Security Policy

## Supported Versions

GearRay is under active early development. Security and serious correctness
fixes are applied to the current `main` branch. Historical engine tags are
preserved as development milestones and are not maintained as separate
supported release lines.

## Reporting a Vulnerability or Serious Issue

Do not open a public issue containing exploit details, private data, or
information that could put users or their development systems at risk.

Use GitHub's private vulnerability reporting for GearRay:

<https://github.com/edenas/GearRay/security/advisories/new>

Include:

- a concise description of the issue;
- affected files, versions, tags, or toolchain components;
- reproduction steps or a proof of concept;
- the potential impact;
- any suggested mitigation;
- whether third-party software is involved.

If private vulnerability reporting is unavailable, open a minimal public issue
requesting a private contact channel without disclosing sensitive details.

Ordinary gameplay bugs, build failures, documentation problems, and
non-sensitive correctness issues may be reported through the public issue
tracker.

The maintainer will acknowledge a private report, assess its scope, coordinate
a fix when appropriate, and credit the reporter unless anonymity is requested.
